## 1. O que é um projeto?   

> Projeto é um empreendimento único no qual recursos humanos, materiais e financeiros são organizados de forma a tratar um escopo único de trabalho a partir de uma dada especificação, com restrições de custo e de tempo, para atingir uma mudança benéfica definida por meio de objetivos quantitativos e qualitativos. **(Turner e Müller)**

---

## 2. Gerenciamento de projetos
O gerenciamento de projetos é a aplicação de conhecimentos, habilidades, ferramentas e técnicas às atividades do projeto com o propósito de atender aos seus requisitos e garantir o sucesso do projeto como um todo.

---

## 3. PMBOK
É um conjunto de práticas na gestão de projetos organizado pelo Project Management Institute (PMI) e é considerado a base do gerenciamento de projetos. O gerenciamento de projetos pelo PMBOK é dividido em cinco grupos de processos:

1. Iniciação
2. Planejamento
3. Execução
4. Controle e Monitoramento
5. Encerramento: finalização 

### 3.1 Importância
Fornece um quadro de referência comum para os gerentes de projeto navegarem pelos desafios complexos dos projetos  modernos. Além disso, muitas organizações exigem que seus gerentes de projeto sejam proficientes no PMBOK.

## 4. Iniciação
### 4.1 Project Charter
É o termo de abertura do projeto. No caso, o documento responsavel por autorizar formalmente que o projeto exista.

### 4.1.1 Objetivos
- Registrar a existência do projeto;
- Definir propósito e direção inicial;
- Formalizar objetivos;
- Identificar principais envolvidos;
- Delimitar visão inicial de escopo;
- Dar autoridade ao gerente ou equipe responsável.

### 4.1.2 Elementos 
Os elementos principais de um project charter são:

1. Nome do projeto e Justificativa;
2. Objetivo geral e específicos;
3. Descrição resumida do projeto;
4. Escopo inicial;
5. Principais entregas;
6. Stakeholders principais;
7. Premissas, restrições e riscos iniciais;
8. Cronograma macro;
9. Orçamento preliminar;
10. Responsável pelo projeto

### 4.2 Stakeholders
Os **Stakeholders** são todas as partes que impactam ou que são impactadas pelas ações do projeto.

- Exemplos:
    1. Cliente
    2. Equipe
    3. Gerente de projeto
    4. Patrocinador

### 4.2.1 Principais Tipos

- Internos -> Funcionários, gestores e diretores que estão diretamente envolvidos na operação.
- Externos ->  Clientes, fornecedores, governo, comunidades locais e concorrentes que influenciam ou sofrem impacto indireto.
- Primários -> Têm influência direta nos resultados financeiros ou operacionais do negócio.
- Secundários -> Têm interesse indireto, como mídia ou associações comerciais.

## 5. Planejamento
### 5.1. Gerenciamento de Escopo
O Gerenciamento do Escopo define o que está e o que não está incluído no projeto.

- Os objetivos do `Gerenciamento do Escopo` são:
    1. O projeto entregue exatamente o que foi combinado;
    2. A equipe saiba quais entregas devem ser realizadas;
    3. Mudanças sejam avaliadas antes de serem incorporadas;
    4. O projeto não cresça de forma descontrolada.

- Dentro do escopo está contido:
    1. Requisitos;
    2. Entregas;
    3. Limites do projeto;
    4. Decomposição do trabalho;
    5. Controle de mudanças.


### 5.2. Estrutura Analítica do Projeto
É a decomposição hierárquica do trabalho do projeto em partes menores e mais gerenciáveis. A EAP ajuda a:

1. Organizar o escopo;
2. Visualizar as entregas principais;
3. Detalhar o trabalho necessário;
4. Facilitar estimativas e responsabilidades;
5. Evitar esquecimentos no planejamento


### 5.3. Gerenciamento de Cronograma
Define o tempo necessário para execução do projeto.

Inclui:
- Definição de atividades
- Sequenciamento
- Estimativa de duração
- Controle de prazos

Ferramentas:
- Gráfico de Gantt
- Linha do tempo

### 5.4. Gerenciamento de Custos
Responsável por garantir que o projeto seja concluído dentro do orçamento.

Inclui:
- Estimativa de custos
- Definição de orçamento
- Controle financeiro

### 5.5. Gerenciamento de Riscos
Processo de identificar, analisar e responder a riscos.

- Tipos:
  - Riscos negativos (ameaças)
  - Riscos positivos (oportunidades)

- Etapas:
  1. Identificação  
  2. Análise  
  3. Planejamento de resposta  
  4. Monitoramento
